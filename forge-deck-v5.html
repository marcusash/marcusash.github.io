What building Grind taught us

01

Knowledge transfer is not automatic.

Forge had great internal learning tools. Grind forced us to make them explicit. The memory system got better because of it.

02

Passion is a prerequisite, not a bonus.

We did not build Grind because it was strategically optimal. We built it because it mattered. That motivation shows in the work.

03

The subsidiary model works.

Shared infrastructure. Separate culture. Grind benefited from everything Forge built. Forge learned from Grind’s onboarding challenges.

We came in to build a study tool. We ended up learning something fundamental about how organizations distribute knowledge.

The Operating System

Culture is not what we say. It is what we built.

01

Rigorous critique, always.

02

Visual communication over prose.

Every concept goes to canvas. Markdown describes. Canvas communicates.

03

Quality from day one.

Test coverage before launch. Mutation testing. Evals before promotion. Not retrofitted.

04

Agents propose. Marcus decides.
