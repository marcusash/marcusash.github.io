GA Handoff Notes

FD implementation priorities for GA. Ordered by user impact vs complexity. Start from the top.
  • 1
    Home screen date grouping and word count badges
    This is the highest-impact change to the current list view. Group entries under "Today / Yesterday / This Week / Earlier" headers. Add word count and mood to each row. Use the existing entries data model, no schema changes needed. Target: render 200 entries without scroll jank using a virtual list or windowed rendering approach in Ink.
  • 2
    Autosave and time-in-editor counter in the editor
    The current editor requires manual save. Add a 10-second idle autosave (debounced). Display X words | Y:ZZ in editor at the bottom left. The time counter starts when the editor opens and pauses when the process loses focus. This removes the biggest anxiety point for new users who forget to save.
  • 3
    Mood picker with F1-F5 keyboard shortcuts
    A five-emoji mood row at the bottom of the editor, navigable by F1-F5. Persists the mood value to the entry record. Show the selected mood on the home list rows. Schema needs a mood column (integer 1-5 or null). Keep the display as emoji rather than labels to save horizontal space in the list.
  • 4
    Full-text search with filter chips
    Wire SQLite FTS5 on the entries table (content and title columns). The search bar on the home screen should activate with / and show results inline, no separate screen required. Filter chips for date range and mood can be a second iteration. Highlight matched substrings in the result preview using a different color (--maize). No network. No external index.
  • 5
    Memory Lane reflection mode
    Query the entries table for records where date matches today's month and day, filtered to prior years and prior months. Display the two most relevant matches above the current entry in a dimmed style. This feature requires no new data, only a date-based query. Add a guard: if no entries exist for that historical date, show a placeholder message rather than an empty section.
  • 6
    Insights panel and Markdown export
    Implement the Insights panel as a slide-in overlay ([I] key). Compute stats (word totals, streak, top tags, mood average) at write time and store in a stats table to avoid scanning all entries on open. For export, start with Markdown only: serialize entries in the selected date range to a single .md file with frontmatter. PDF and email targets are follow-on. Do not block the Insights panel on export completion.
