Design principles for canvas-ready HTML

Every HTML artifact we build should follow these rules so it works in both browser and canvas:

PrincipleRuleWhy
Dark background enforcedbackground:#0A0A0F on html + bodyApp mode shows OS bg flash on open if not set
Viewport-relative layoutUse vw/vh, not px for top-level sizingCanvas window is resizable; layout must adapt
No external network depsNo CDN fonts, no remote scriptsCanvas may be offline; file:// has no network
Canvas meta tag<meta name="forge-canvas" content="ready">Electron can read this to configure window
No scrollbars by defaultoverflow:hidden on body; scroll inside panelsCanvas is a viewport, not a document
Prefer system fontsSegoe UI Variable + Cascadia Code (already on machine)No font load flash in file:// context
