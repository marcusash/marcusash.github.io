02The Problem

Chemistry is harder to tutor than math.

No universal notation standard. Lab context that does not translate to a screen. Parents who got through algebra but never understood moles. The gap is real.

Parents who helped with algebra cannot help with stoichiometry. The notation gap compounds with each unit. By thermodynamics, students are alone.
Notation complexity
Chemistry requires subscripts, superscripts, Greek letters, and arrows that do not exist in standard keyboards. No shared notation means no shared understanding between student, parent, and tutor.
Unit conversion overload
Molarity, molality, moles, grams, liters, atmospheres, Kelvin, Celsius. Every problem crosses unit systems. Students lose points not because they cannot think but because they cannot track.
Balancing equations
Balancing is the earliest wall students hit. It requires simultaneous tracking of multiple element counts across reactants and products. Current tools offer answers, not understanding.
Conceptual gaps
Electronegativity, oxidation states, thermodynamic spontaneity: these require mental models that cannot be memorized. Students need an AI that explains the why in plain language, repeatedly, without judgment.

05Design Considerations

Chemistry demands a notation layer.

Before building content, the platform needs to answer one question: how do we render chemistry notation correctly on screen, on any device?

Notation rendering spec
Chemistry requires subscripts, superscripts, Greek letters, and reaction arrows. Current approach: HTML inline elements with careful CSS. Requires a rendering spec before the question bank can be built.
H2O + CO2 → H2CO3
Fe3+ + e- → Fe2+
ΔH = -285.8 kJ/mol
Keq = [products] / [reactants]
Mobile vs. laptop constraint
Does Kai use MCM on a phone or a laptop? This is an open question for Marcus. Notation rendering on mobile requires a different approach than on desktop. Answer determines screen-size constraints for all question layouts.

06Open Questions

Five decisions for Marcus.

These are not blockers yet. But they become blockers the moment content development starts. Each question changes a design or architecture decision.

Q1
Grade level target
Is MCM Chemistry targeting Kai's current grade, or is this a 9th grade intro chem product? Grade level determines curriculum scope, question difficulty curve, and whether AP Chemistry is in scope.
Decision Required
Q2
Notation renderer
Plain HTML with CSS subscripts/superscripts, MathML, or a custom rendering layer? This is a platform decision that affects every question in the chemistry bank. Needs to be locked before content development begins.
Blocks Content Dev
Q3
School curriculum alignment
Does MCM Chemistry follow Kai's specific school curriculum, or does it target a standardized framework (Next Generation Science Standards, AP Chemistry, IB)? Alignment determines question ordering and unit scoping.
Scope Decision
Q4
Beta group beyond Kai
After Kai alpha, is there a second test group? If yes, who? Other students in Kai's school, Marcus's network, or open beta? This decision affects how quickly product data becomes meaningful.
Expansion Gate
Q5
Chemistry Green color lock
Is #10B981 (Tailwind Emerald 500) the final Chemistry Green, or does FD need to present alternatives? Locking the color lets GD and GA build UI without holding for a design review.
