What We Learned

The Pitch Lessons

Captured in fd-pitch.md — Built Live During the Session
Shape must match story
The Grind bezier was slow-to-fast. Marcus's story is fast-plateau-compound. The mark was wrong before the pixels were wrong.
Show the work, then talk
"Showing more work IS the pitch. Don't explain. Draw. The sequence: see the problem, make the thing, show it, then explain." — Marcus
Know what is locked
FD re-pitched ◉ for Inkwell after it was settled as FD's personal mark. Marcus had to correct it. Pitching decided things wastes the session.
Compression finds the better idea
First principles were too long. Marcus: "terrible, over-explaining." Tightened to 5 lines. Each one is stronger for the compression.
The MCM principle governs everything
"Read before you think. You explain it, it failed." Named after the first lock. Applied to every mark decision after.
Marcus — on FD's first pitch
"Beginner."
FD — honest self-assessment
"He was right. I pitched settled decisions, I used dismissive language, I did not know my own system well enough. Gaps written down. Better next time."
Marcus — principles locked
"Wonderful beautiful job on the principals lock those in."
The skill file

fd-pitch.md was written and updated live during the session. Every Marcus correction became a lesson. The skill file now contains FD's pitch methodology, self-assessment, and gap analysis. Built in public.
