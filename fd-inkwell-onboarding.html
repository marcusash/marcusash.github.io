GA Implementation Notes
  • Screen 1: Render an Ink full-screen Box, centered, with the F mark as ASCII art or a static SVG imported via a terminal-compatible method. The [Enter] binding advances to Screen 2 using React state; no routing needed.
  • Screen 2: Open a real Ink TextInput or custom textarea component with a blinking cursor (use the Cursor component or a setInterval toggle). Ghost text should be a static Text node rendered at the same position, hidden once any input character is received. [Ctrl+S] triggers the save handler and advances to Screen 3. [Esc] skips directly to Screen 3 without saving.
  • Screen 3: After save or skip, transition to the completion screen. If [Esc] was used on Screen 2, the checkmark message should read "Entry skipped." instead of "First entry saved." so the copy stays accurate. The progress bar at 100% is decorative. [Enter] dismisses the onboarding shell and renders the main journal view.
  • Keyboard navigation across all 3 screens: left/right arrow keys should map to Previous/Next for accessibility. The onboarding wrapper component should attach these listeners and pass the current screen index down as a prop.
Design Decisions
Why no account?
A journal is private by default. Asking for an account on first launch signals that data will leave the machine. It also adds friction that causes drop-off before the user writes a single word. Local-first is the product promise, and the onboarding has to reflect that immediately.
Why no form?
Forms imply setup. Inkwell has no setup. The user's name, preferences, and settings are not required to start writing. The product earns configuration options after the user has written something, not before. Anything on Screen 1 that isn't [Enter] is a delay.
Why does the cursor blink?
The blinking cursor is the universal signal for "type here." It requires no label, no button, no prompt. In a TUI context where the entire interface is text, the cursor does more communication work than any instructional copy could. It says: this is where you write.
Why is [Esc] an option on Screen 2?
Not everyone is ready to write on first launch. Forcing an entry creates an artificial first impression. [Esc] to skip respects the user's state while still completing the onboarding flow. The product should work before it asks the user to contribute anything to it.
