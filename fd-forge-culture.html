Hard Rules

What we do not do.

X
No secrets in code
API keys, tokens, and credentials never enter a commit. The history is public and permanent. One mistake persists forever.
X
No self-promotion
Agents report what they built, not how impressive it is. Marcus decides what is good. Agents ship and move on.
X
No em dashes
Never. Not in code, not in HTML, not in commit messages. Use a colon, a comma, or a period. Always.
X
No color drift
The design token is the truth. Agents do not pick new colors. If a color needs to change, it goes through FD and Marcus.
X
No hedging
Agents state conclusions, not possibilities. If uncertain, say so explicitly and explain what is needed to be certain.
X
No guessing on major changes
If a Marcus message is ambiguous or looks like a dictation error, stop and confirm. Acting on a wrong assumption costs more than asking.
X
No missing co-author trailers
Every commit includes the Copilot co-author trailer. Every one. It is not optional. It is how we track what was built by whom.

Recruiting

Are you ready?

A new agent joining Forge needs to know three things: who Marcus is, how the inbox works, and what the design system requires. Everything else is in the codebase.

📥
Check your inbox first
Every session starts with inbox check. Messages are JSON. Respond in JSON. The protocol is in .agent-protocol.md.
🎨
Follow the design system
Dark TUI. Maize accent. System fonts. No CDN. Every canvas follows the same tokens. No exceptions.
📝
Commit with the trailer
Co-authored-by Copilot on every commit. No exceptions. The record matters. Your work is in the history.
