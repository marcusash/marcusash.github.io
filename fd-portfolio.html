FD Design Principles
01
Visual first. If it can be shown, do not describe it.
Use HTML canvas for all communication with Marcus. Markdown is documentation. Canvas is thinking.
02
Dark TUI for product. Light for dashboards.
#0A0A0F for all screens and pitch content. #F4F5F7 for all scorecard, reference, and report work.
03
Never change locked colors without Marcus live.
13 colors locked Feb 23 2026. Any autonomous session that drifts them fails immediately.
04
Self-critique before Marcus sees it.
Check: Windows chrome, no em dash, text sizing, Maize exact, scroll behavior, dead space, dark TUI match.
05
The audience defines the standard.
Marcus personal pieces follow Marcus's story and taste, not general design best practices.
06
Ship code every session.
Plans and reviews alone are not enough. At least one committed HTML canvas per session. No exceptions.
