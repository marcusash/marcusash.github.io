05 — Rules
Headline tracking
Headlines at 1.5rem+ use -0.02em to -0.03em tracking. Never positive tracking on large type.
Mono for data
Cascadia Code is only for: hex colors, code, numeric data, status labels, keyboard shortcuts, and timestamps.
Line height
Body: 1.6. Headlines: 1.1 to 1.2. Labels/mono: 1.4. Never below 1.0.
Max measure
Body copy: 60-72 characters per line. Labels: no limit. Headlines: no limit.
All caps
All caps is only for: eyebrows/labels in mono at 0.6rem or below. Never for body text or headlines.
Em dashes
NEVER use em dashes (— or –). Use a colon, comma, or period. Hard rule, no exceptions.
06 — Do / Don't
Do
Use Segoe UI Variable for all UI prose
Use Cascadia Code for all labels, hex values, and status text
Use letter-spacing: -0.02em on headlines above 1.5rem
Scale text with clamp() for responsive layouts
Keep body weight at 400 for readability
Use #E8E8F0 for primary text on dark backgrounds
Don't
Use CDN fonts (Google Fonts, Typekit, etc.)
Use all-caps on text above 0.7rem
Apply positive letter-spacing to large headlines
Mix Segoe and Cascadia in the same text element
Use em dashes, ever
Use pure #FFFFFF or #000000 for text (use #E8E8F0 and #0A0A0F)
